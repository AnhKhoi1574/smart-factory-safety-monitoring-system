# Behavior Datasets Preprocessing Pipeline

This folder contains the raw data, extraction scripts, and notebooks to pre-process pose-video datasets for behavior classification models (e.g., XGBoost, LightGBM, Random Forest).

## Directory Structure

```text
behavioral-detection/
  raw_data/                  Raw pose-video export sub-directories:
    fall/                    - keypoint_windows.npz, annotations.jsonl, manifest.json
    no-fall/
    run/
  features_extracted_data/   NPZ splits generated by extract_engineered_features.py:
    train_split.npz          - 70% train split
    val_split.npz            - 15% validation split
    test_split.npz           - 15% test split
  cleaned_features_data/     Balanced and cleaned datasets exported by behavior_preprocess.ipynb:
    train_cleaned.npz        - Balanced training set (1,200 samples/class)
    val_cleaned.npz          - Cleaned validation set
    test_cleaned.npz         - Cleaned test set
  extract_engineered_features.py  Features extraction and stratified group splitting script
  behavior_preprocess.ipynb       Jupyter notebook to clean, balance, and transform splits
  preprocessing_guide.md          Mathematical breakdown and details of motion features
```

---

## Preprocessing Workflow

### Step 1: Prepare Raw Data
Ensure your raw pose-video datasets (containing `keypoint_windows.npz`, `annotations.jsonl`, and `manifest.json`) are located in the `raw_data/` directory:
* `raw_data/fall/`
* `raw_data/no-fall/`
* `raw_data/run/`

### Step 2: Run Feature Extraction & Split
Execute the standalone feature extraction script from the repository root:
```powershell
python behavioral-detection/extract_engineered_features.py
```
This script:
1. Calculates **143 physical movement descriptors** (torso angles, speed decimated time-series, stillness, tracking quality) for all windows.
2. Normalizes hip-ankle vertical ratio inaccuracies by clipping them to `1.0`.
3. Performs a **leakage-free stratified group split** to partition windows by their source `video_id` while keeping the class ratio (approx 72.5% others, 7.6% running, 19.9% falling) identical across splits.
4. Outputs `train_split.npz`, `val_split.npz`, and `test_split.npz` into `features_extracted_data/`.

### Step 3: Resample and Clean
Open and execute the cells inside the Jupyter notebook:
```powershell
# Open Jupyter lab/notebook from repository root
jupyter notebook behavioral-detection/behavior_preprocess.ipynb
```
The notebook:
1. Loads the splits from `features_extracted_data/`.
2. Resolves class imbalance in the training split by performing random undersampling on `others` and random oversampling on `running` and `falling` (balancing them at 1,200 samples per class).
3. Applies a square-root transformation to skewed speed and acceleration features to improve model performance.
4. Exports `train_cleaned.npz`, `val_cleaned.npz`, and `test_cleaned.npz` directly to `cleaned_features_data/` for direct ingestion by machine learning models.
